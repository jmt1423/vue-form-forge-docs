Thank you for your interest in contributing to Vue Form Forge! I welcome contributions from the community and would
love to see how far this can go.

## Code of Conduct

By contributing to this project, you agree to abide by our code of conduct. Please treat everyone
with respect and and don't be condescending.

## Getting Started

### Prerequisites

- Node.js (v16+)
- npm


<Aside type="note">
    This is a monorepo setup and all the code is in the packages/\* directory.
    The main vue app has the form builder as a dependency in the package.json file,
    and the code for the builder is in packages/form-builder/
</Aside>

### Setting Up the Development Environment


<Steps>
1. **Fork the repository** on GitHub

2. **Clone your fork** locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/vue-form-forge.git
   cd vue-form-forge
   ```
3. **Install dependencies**:
   ```bash
   npm install
   ```
4. **Create a new branch** for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```
</Steps>

## üîß Code Contributions

### Pull Request Process

<Steps>
    1. **Ensure your code follows our standards**:
        - Write clear, readable code
        - Include appropriate comments
    2. **Test your changes**:
        - Add tests for new features (the library uses Vitest and Playwright)
        - Ensure all existing tests pass
        - Test in different Vue and Node versions if applicable
    3. **Update documentation**:
        - Update README.md if needed
        - Update this documentation site if necessary
    4. **Commit your changes**:
        ```bash
        git add .
        git commit -m "feat: add new form validation feature"
        ```
    5. **Push to your fork**:
        ```bash
        git push origin feature/your-feature-name
        ```
    6. **Create a Pull Request**:
        - Use a clear and descriptive title
        - Write a decent description of your changes
        - Link any related issues
</Steps>

### Commit Message Convention

- `feat:` for new features
- `bugfix:` for bug fixes
- `style:` for formatting changes
- `refactor:` for code refactoring
- `test:` for adding tests
- `chore:` for maintenance tasks
